# maze_ai — 经典算法 AI 玩家（任务二）

确定性、可复现、无需训练 / 无需 GPU 的迷宫探险 AI 玩家。直接读取**官方输入格式**
（`maze` / `B` / `PlayerSkills` / `minRouds` / `CoinConsumption`），对*任意*迷宫给出
接近最优的资源收集 + BOSS 战 + 抵达终点方案，并可视化全过程。

> 设计取向：本课程是**算法**课，评分点是经典算法范式（分治 / 动态规划 / 贪心 / 回溯 /
> 分支限界）+ 伪代码 + 复杂度分析。确定性算法对交叉测试中**未知迷宫**天然泛化，且每步可解释——
> 这正是替代此前"训练效果差、无法泛化"的神经网络（DQN/LoRA）方案的原因。

## 关键事实：迷宫是一棵树

“完美迷宫”=无孤立区域且任意两格之间存在**唯一通路** ⟺ 可行走格子构成一棵**树**。
因此任意两点间距离与路径唯一，一次 BFS 即可得到某源点到所有格子的最短距离与路径
（父指针重建）。这把资源收集化简为树上的**定向越野（orienteering）**问题。

## 算法与复杂度

记 `V` = 可行走格子数，`m` = 金币数，`s` = 技能数，`Hmax` = 单个 BOSS 最大血量，
`n` = BOSS 个数。

### 1. 最短路径 / 全源距离（BFS）— `grid.py`
- **范式**：BFS（分支限界/BFS 的基础）。
- 对每个“路标”（起点、各金币、BOSS、终点）做一次 BFS，得到其到所有格子的距离与唯一路径。
- **时间** `O((m+3)·V)`，**空间** `O((m+3)·V)`。

### 2. 资源收集最优路线（位掩码动态规划，Held–Karp 风格）— `planner.py`
- **范式**：动态规划。
- `dp[mask][i]` = 从起点出发、恰好采集 `mask` 这组金币、止于金币 `i` 的**最少步数**。
  - 初始：`dp[{i}][i] = dist(start, i)`
  - 转移：`dp[mask∪{j}][j] = min_i dp[mask][i] + dist(i, j)`
- 对每个子集 `mask` 取“收尾代价最小”的终点（`dp[mask][i] + dist(i, BOSS)` 最小），
  得到该子集的一条候选路线，再以 `→ BOSS → 终点` 收尾。
- 每条候选路线沿真实树上路径**精确结算**（顺路捎带的金币 / 陷阱、陷阱只触发一次、
  BOSS 在最优技能下 0 金币代价），取 `资源/步数` 比值最高者。
- **时间** `O(2^m · m^2)`（DP）+ 候选精确结算；`m ≤ 15` 用 DP，更多金币退化为贪心插入启发式。
- **空间** `O(2^m · m)`。
- 已用**完整 (子集×终点) 枚举**交叉验证：本实现得分与之逐例相等（gap = 0）。

### 3. BOSS 战最少回合 + 最优技能序列（分支限界）— `boss.py`
- **范式**：分支限界（状态空间 BFS）。
- 状态 `(当前BOSS下标, 当前BOSS剩余血量, 各技能冷却向量)`；每回合用一个未冷却技能 = 一次状态转移，代价 1。
- BFS 按回合数非降扩展，首次到达“全部 BOSS 击破”即**最少回合数**；父指针重建**最优技能序列**。
  访问集合即剪枝（被支配状态丢弃）。
- 再按回合上限 `minRouds` 与 `CoinConsumption` 结算复活/金币消耗（最优技能下通常 0 复活）。
- **时间/空间** `O(n · Hmax · ∏(cd_i+1))`（状态数很小）。
- 已用**暴力 DFS**交叉验证：300 个随机配置最少回合数全部一致。

### 4. 实时 3×3 资源拾取（贪心）— `greedy3x3.py`
- **范式**：贪心。仅凭 3×3 视野，每步移向“单位距离收益”最高的相邻格（金币 `+50`、陷阱 `-30`）；
  无正收益时走访问最少的格子做探索，避免环路。
- **时间** `O(步数)`，**空间** `O(V)`。

## 资源/金币经济
```
剩余资源（金币） = 50·采集金币数 − 30·触发陷阱数 − CoinConsumption·复活次数
```
拾取一个金币格（`G`）得 **50 金币**，踩一次陷阱（`T`）扣 **30 金币**（每个陷阱只触发一次），
BOSS 每复活一次扣 **`CoinConsumption`（=5）金币**。在最优技能序列下，本组 BOSS 在回合上限
（`minRouds`）内被击破，复活次数通常为 0。
